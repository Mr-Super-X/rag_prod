## 1. 部署全景图

部署后的架构长这样：

```
用户浏览器（公网）
  │
  ▼
┌──────────────────────────────────────┐
│  Nginx (HTTPS 终结 + 静态文件服务)     │
│  · rag.example.com → 前端页面          │
│  · rag.example.com/api/* → 后端 3000   │
└──────────────┬───────────────────────┘
               │
┌──────────────▼───────────────────────┐
│  PM2 守护的 Node.js 进程 × 2          │
│  Fastify API (localhost:3000)         │
└──────────────┬───────────────────────┘
               │
    ┌──────────┼──────────┐
    ▼          ▼          ▼
┌──────┐ ┌──────┐ ┌──────────┐
│  PG  │ │Redis │ │  Ollama  │
│ :5432│ │:6379 │ │  :11434  │
└──────┘ └──────┘ └──────────┘
```

简化版：一台服务器跑全部东西。后端用 PM2 保活，前端编译成静态文件交给 Nginx，Nginx 顺便做 HTTPS 和反向代理。

## 2. 购置服务器

### 最低配置

| 配件 | 规格 | 用途 |
|------|------|------|
| CPU | 4 核 | 业务逻辑 + 向量搜索 |
| 内存 | 16GB | 系统 4G + PG 2G + Redis 1G + 模型 5G + 余量 |
| 硬盘 | 100GB SSD | 系统 20G + 模型 10G + 文档 + 日志 |
| 带宽 | 5Mbps | 10 人同时用够，20 人建议 10Mbps |

### 推荐云厂商

| 厂商 | 推荐机型 | 月费参考 |
|------|---------|---------|
| 阿里云 ECS | ecs.c7.xlarge（4C 16G） | ¥500-700 |
| 腾讯云 CVM | SA5.LARGE8（4C 16G） | ¥400-600 |
| AWS EC2 | t3.xlarge（4C 16G） | $120-150 |
| AutoDL |  RTX 3090 方案 | ¥900/月 |

> 如果你有 NVIDIA GPU（A10/3090 以上），选 AutoDL GPU 云服务器，推理速度从分钟级变成秒级。无 GPU 就用普通云服务器跑 CPU 推理，也能用，只是慢。

### 操作系统选型

选 **Ubuntu 22.04 LTS**（最广泛使用、教程最多、出问题了 Google 最快找到答案）。

## 3. 服务器环境搭建

以下所有命令在服务器终端执行。

### 3.1 更新系统

```bash
sudo apt update && sudo apt upgrade -y
```

### 3.2 安装 Node.js

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo bash -
sudo apt install -y nodejs
node -v   # 确认 >= 20
```

### 3.3 安装 Docker

```bash
curl -fsSL https://get.docker.com | sudo bash
sudo usermod -aG docker $USER   # 免 sudo 运行 docker
# 退出重新登录让权限生效
```

安装 Docker Compose 插件：

```bash
sudo apt install -y docker-compose-plugin
docker compose version   # 确认可用
```

### 3.4 安装 Nginx

```bash
sudo apt install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx
```

浏览器访问 `http://你的服务器IP`，看到 Nginx 欢迎页就 OK。

### 3.5 安装 Ollama

```bash
curl -fsSL https://ollama.com/install.sh | sudo sh
# 安装脚本会自动注册 systemd 服务并启动，无需手动 ollama serve
ollama pull qwen2.5:7b     # 拉 LLM 模型
ollama pull bge-m3         # 拉 Embedding 模型
```

> 如果 `ollama pull` 报 `connection refused`，手动启动：`sudo systemctl start ollama`

> 有 GPU 的话，Ollama 会自动检测并使用。`ollama serve` 在 /etc/systemd/system/ollama.service 会自动注册为系统服务。

### 3.6 安装 Git

```bash
sudo apt install -y git
```

## 4. 上传项目代码

### 方式一：Git 仓库（推荐）

先在 GitHub/GitLab 上建私有仓库，把项目 push 上去，然后服务器拉取：

```bash
git clone <你的仓库地址> /opt/rag_prod
cd /opt/rag_prod
```

之后每次更新代码：

```bash
cd /opt/rag_prod
git pull
```

### 方式二：scp 直接上传

```bash
# 在你本地电脑执行（替换 /path/to/rag_prod 为你的项目路径）
scp -r /path/to/rag_prod 用户名@服务器IP:/opt/rag_prod
```

## 5. 启动服务

### 5.1 创建 .env

```bash
cd /opt/rag_prod
```

手动创建 `.env` 文件：

```
DATABASE_URL=postgresql://raguser:ragpass@localhost:5432/ragdb
OLLAMA_URL=http://localhost:11434
REDIS_URL=redis://localhost:6379
JWT_SECRET=替换为随机字符串
UPLOAD_DIR=./data/uploads
```

> `JWT_SECRET` 不可用默认值。生成随机串：`openssl rand -base64 32`

### 5.2 启动基础设施

```bash
docker compose up -d
```

验证：

```bash
docker ps   # 应该看到 postgres 和 redis 两个容器
```

> **注意**：§3.5 已原生安装了 Ollama（监听 localhost:11434），所以 Docker 里不用再跑一个 Ollama。如果直接 `docker compose up -d`（用的是 `docker-compose.yml`，里面包含 ollama 服务），两个 Ollama 会端口冲突。
>
> **解决方法**：在服务器上复制一份不含 ollama 的编排文件来用：
> ```bash
> docker compose -f docker-compose.mac.yml up -d
> ```
> `docker-compose.mac.yml` 里只有 PostgreSQL 和 Redis，刚好适用。

### 5.3 初始化数据库

```bash
# 如果 npm install 报错（原生模块编译失败），先装 build tools：
# sudo apt install -y build-essential python3

npm install
npm run db:generate    # 生成数据库迁移文件
npm run db:migrate     # 执行迁移，创建表
```

### 5.4 启动后端

```bash
npm run dev   # 先开发模式验证
```

测试 API：

```bash
curl http://localhost:3000/api/health
# 应该返回 {"status":"ok",...}
```

确认无误后 `Ctrl+C` 停掉。下一步用 PM2 正式运行。

## 6. 配置域名与 HTTPS

### 6.1 域名解析

在域名服务商（阿里云/腾讯云/Cloudflare）添加一条 A 记录：

```
类型: A
主机记录: rag（或你想要的子域名）
记录值: 你的服务器 IP
TTL: 600
```

### 6.2 申请 SSL 证书

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d rag.你的域名.com
```

按提示输入邮箱，同意协议。Certbot 会自动修改 Nginx 配置，加好 HTTPS。

### 6.3 证书自动续期

```bash
sudo certbot renew --dry-run   # 测试自动续期
```

Certbot 安装后会自动加定时任务，无需手动续。

## 7. 进程守护（PM2）

`npm run dev` 在终端关了进程就死。用 PM2 保活。

### 7.1 安装

```bash
npm install -g pm2
```

### 7.2 构建后端

```bash
cd /opt/rag_prod
npm run build      # 编译 TypeScript → dist/
```

### 7.3 配置启动文件

项目已包含 `ecosystem.config.cjs`（`.cjs` 是因为项目用 ESM 模块，PM2 配置需要 CommonJS 格式）。如需调整进程数，编辑 `instances` 值（建议设为 CPU 核心数）。

### 7.4 启动

```bash
mkdir -p logs
pm2 start ecosystem.config.cjs
pm2 save            # 保存进程列表
pm2 startup         # 开机自启（按提示执行输出的命令）
```

### 7.5 常用命令

```bash
pm2 list            # 查看进程状态
pm2 logs rag-api    # 查看日志
pm2 restart rag-api # 重启
pm2 stop rag-api    # 停止
```

## 8. 前端生产部署

### 8.1 构建

```bash
cd /opt/rag_prod/client
npm install
npm run build       # 产物在 dist/ 目录
```

### 8.2 配置 Nginx 静态服务

编辑 `/etc/nginx/sites-available/rag`：

```nginx
server {
    listen 80;
    server_name rag.你的域名.com;

    # 前端静态文件
    root /opt/rag_prod/client/dist;
    index index.html;

    # Vue Router history 模式
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 文件上传大小限制
    client_max_body_size 50M;
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/rag /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default   # 去掉默认页
sudo nginx -t                               # 检查配置
sudo systemctl reload nginx
```

### 8.3 申请 HTTPS（域名解析后）

```bash
sudo certbot --nginx -d rag.你的域名.com
```

完成后 Nginx 配置会被 certbot 自动加上 SSL 相关配置。

## 9. Nginx 反向代理

上一步的 Nginx 配置已经包含了反向代理部分（`location /api/`）。核心作用：

```
请求: https://rag.example.com/api/auth/login
  ↓
Nginx 解密 HTTPS → 转发到 http://localhost:3000/api/auth/login
  ↓
Fastify 处理 → 返回 JSON
  ↓
Nginx 加密 → 返回给浏览器
```

用户全程只和 Nginx 交互，后端不需要暴露到公网。

## 10. 数据备份

### 10.1 PostgreSQL 备份

```bash
# 创建备份脚本 /opt/backup-pg.sh
#!/bin/bash
BACKUP_DIR=/opt/backups
mkdir -p $BACKUP_DIR
docker exec rag_prod-postgres-1 pg_dump -U raguser ragdb > $BACKUP_DIR/ragdb_$(date +%Y%m%d).sql
# 保留最近 7 天的备份
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete
```

加入 crontab 每日凌晨 3 点备份：

```bash
chmod +x /opt/backup-pg.sh
crontab -e
# 添加: 0 3 * * * /opt/backup-pg.sh
```

### 10.2 LanceDB 和上传文件备份

```bash
# 追加到 backup-pg.sh
tar -czf $BACKUP_DIR/lancedb_$(date +%Y%m%d).tar.gz /opt/rag_prod/data/lancedb/
tar -czf $BACKUP_DIR/uploads_$(date +%Y%m%d).tar.gz /opt/rag_prod/data/uploads/
```

### 10.3 恢复

```bash
# PG 恢复
docker exec -i rag_prod-postgres-1 psql -U raguser ragdb < ragbd_20260526.sql

# LanceDB + 文件恢复
tar -xzf lancedb_20260526.tar.gz -C /opt/rag_prod/data/
```

## 12. 常见问题

### Q: 服务器内存不够，模型加载失败

**症状**：Ollama 启动后 `ollama pull` 成功但问答返回 500。

**解决**：Qwen2.5-7B Q4 需要 ~5GB 可用内存。检查：

```bash
free -h              # 看 available 是否 > 5GB
```

如果不够：加内存条，或者换成更小的模型 `qwen2.5:3b`（~2GB）。

### Q: 文档上传后一直"处理中"

**症状**：上传成功后文档状态永远不到"就绪"。

**排查**：

```bash
pm2 logs rag-api | grep -i error
```

大概率是 Ollama 挂了或者 Embedding 超时：

```bash
curl http://localhost:11434/api/tags   # 确认 Ollama 在线
```

重启 Ollama：

```bash
sudo systemctl restart ollama
```

### Q: 外网访问 502

**症状**：浏览器打开域名显示 Nginx 502。

**排查**：后端是否在跑 `pm2 list`。如果后端正常，检查 Nginx 错误日志：

```bash
sudo tail -f /var/log/nginx/error.log
```

### Q: HTTPS 证书过期

```bash
sudo certbot renew
sudo systemctl reload nginx
```

正常情况 certbot 会自动续。如果没自动续，手动执行上面的命令。

### Q: 磁盘快满了

```bash
# 查看谁占空间
du -sh /opt/rag_prod/data/*

# 清理旧的上传文件
find /opt/rag_prod/data/uploads -mtime +30 -delete

# 清理 Docker 垃圾
docker system prune -a
```
