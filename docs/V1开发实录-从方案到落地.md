## 一、起点：一份架构文档

2026-05-26，基于之前撰写的《企业级RAG知识库架构设计与落地》文档，决定将其落地为真实可用的系统。

核心约束：
- **开发者**：独立开发，非产品经理
- **场景**：通用多知识库平台，先供 5-20 人技术团队试用
- **硬件**：Intel 核显 + NPU，无 NVIDIA 独立显卡，只能 CPU 推理
- **技术栈**：Node.js/TypeScript（用户熟悉），拒绝 Python

## 六、踩坑记录

### 坑 1：Docker 镜像源全失效

**现象**：`docker-compose up -d` 不断报 `dial tcp: lookup hub-mirror.c.163.com: no such host`

**排查**：逐个测试 6 个配置的镜像源，结果：
- `hub-mirror.c.163.com` — 域名已不存在
- `mirror.baidubce.com` — 域名已不存在
- `d2bgx7ku.mirror.aliyuncs.com` — 403 过期
- `docker.mirrors.dg.com` — 仅内网可用

**解决**：清空 `registry-mirrors`，配 HTTP 代理走外网

### 坑 2：WSL2 GPU 直通失败

**现象**：`nvidia-container-cli: WSL environment detected but no adapters were found`

**原因**：用户是 Intel 核显 + NPU，根本没有 NVIDIA 显卡

**解决**：放弃 GPU，CPU 推理（文档处理 1-2 分钟，问答 30-60 秒）

### 坑 3：Qdrant 镜像不存在于国内源

**现象**：`docker.io/qdrant/qdrant:latest: not found`

**原因**：阿里云镜像源未同步 qdrant 仓库

**解决**：向量库从 Qdrant 切换为 LanceDB（嵌入式，零 Docker 依赖）

### 坑 4：LanceDB 创建空表失败

**现象**：`At least one record or a schema needs to be provided`

**原因**：`createTable(name, [])` 不允许空数据

**解决**：不在插入前预建表；首次 `insertVectors` 时用实际数据创建表

### 坑 5：LanceDB 向量列未被识别

**现象**：`No vector column found to match with the query vector dimension: 1024`

**原因**：`embedder.ts` 用了 `data.embedding`（单数），Ollama 返回的是 `data.embeddings`（复数数组），导致向量全是 undefined，LanceDB 丢弃了该列

**解决**：纠正字段名为 `embeddings`，且改为一次调用传多个 text

### 坑 6：RRF 融合分数误判检索兜底

**现象**：检索有结果但始终返回"未找到相关内容"

**原因**：RRF 融合分数（0.01-0.05）被拿去和余弦相似度阈值（0.5）比较

**解决**：改为 `sources.length === 0` 才触发兜底

### 坑 7：文档上传同步阻塞

**现象**：50KB 文件上传后页面卡住 98 秒无响应

**原因**：`processDocument` 是同步 await，Embedding 在 CPU 上极慢

**解决**：拆分为二——`processDocument` 先创建数据库记录立即返回，`processDocumentAsync` 在后台执行解析+Embedding+索引

### 坑 8：前端删除文档后不刷新

**现象**：DELETE 返回 204，但列表不更新

**原因**：204 无响应体，`api.delete()` 调 `res.json()` 报错，catch 静默吞掉

**解决**：处理 204 状态码，不解析 body

### 坑 9：Swagger 无请求体输入框

**现象**：Swagger UI 页面"Try it out"后只显示"No parameters"

**原因**：Fastify 路由未声明 `schema: { body: ... }`

**解决**：所有 POST/PATCH 路由加上 JSON Schema

### 坑 10：npm 包版本号屡次不匹配

**现象**：`npm install` 连续失败 3 次：
- `@fastify/swagger@^10.0.1` — no matching version
- `@fastify/jwt@^9.7.0` — no matching version
- `pdf-parse@^1.1.1` — 最新是 2.4.5

**原因**：凭记忆写的版本号，未验证 npm registry 真实版本

**解决**：逐个 `npm view <package> version` 验证，修正为真实存在的版本范围（如 `@fastify/swagger` 最新 9.7.0 而非 10.x）

**教训**：CLAUDE.md 里写了"绝对禁止凭记忆写出 npm 包名"，这次犯了。后续所有 package.json 依赖都先 `npm view` 确认。

### 坑 11：pdf-parse v1 导入即崩溃

**现象**：`import("pdf-parse")` 时报 `ENOENT: no such file or directory, open 'test/data/05-versions-space.pdf'`

**原因**：pdf-parse v1.1.1 在模块初始化时会尝试读取测试文件，该文件在项目里不存在

**解决**：切到 pdf-parse v2（ESM），改用 `PDFParse` 类；同时做降级兼容——try v2 ESM API，失败则回退 v1 CJS API

### 坑 12：@node-rs/jieba 多轮尝试均失败

**现象**：依次尝试了三种 API 均失败：
- `import { cut }` — `Property 'cut' does not exist`
- `new Jieba().cut()` — 分词结果全是单字
- `new Jieba().loadDict()` — `Get TypedArray info failed`
- `Jieba.withDefaultDict()` — `is not a function`

**原因**：@node-rs/jieba v2.x 的 Node.js API 与 Rust 核心的互操作在 Windows 环境下不稳定；`loadDict` 需要特定格式的 ArrayBuffer，传入空 Uint8Array 不合法

**解决**：退而求其次——中文使用字符级 unigram + bigram 组合作为 Tokenizer。精确度不如 jieba 分词，但比纯向量搜索多了一层关键词召回。同时保留 jieba 加载路径，如果未来版本修复可自动启用。切换成本：零（同一接口，tokenize 函数内部无感切换）

### 坑 13：Docker 代理配置踩坑

**现象**：配了 `proxies` 字段（Docker Engine 配置），但拉镜像仍然直连失败，提示"has no HTTPS proxy"

**原因**：Docker Desktop 有两处代理配置，只改了其中一个：
- Settings → Docker Engine → JSON 中的 `proxies` 字段（容器内代理）
- Settings → Resources → Proxies（Docker Desktop 自身的代理，用于拉镜像）

拉镜像走的是后者，只配前者无效。

**解决**：两处都配置 HTTP/HTTPS 代理，且指向正确的代理端口

### 坑 14：Ollama 大镜像拉取频繁中断

**现象**：`docker pull ollama/ollama:latest` 每次到 86 秒左右断开，需连续执行 7 次才下载完 1.6GB 镜像

**原因**：网络代理对长连接不稳定，大文件传输中途断开

**解决**：多次重试，Docker 支持断点续传，每次从中断处继续

### 坑 15：Windows curl 中文 JSON Content-Length 计算错误

**现象**：Bash 中用 curl 发中文 JSON (`-d '{"name":"测试知识库"}'`) 返回 `Request body size did not match Content-Length`

**原因**：Windows 版 curl 对 UTF-8 中文字符的字节数计算不准

**解决**：先将 JSON 写入文件 `echo '...' > /tmp/body.json`，再用 `-d @file` 方式发送

### 坑 16：用户反馈驱动的小迭代

开发过程中根据用户即时反馈做的调整：

| 反馈 | 改动 |
|------|------|
| "Swagger 页面上没找到填参数的地方" | 给所有路由加 JSEND Schema，让 Swagger 生成输入框 |
| "Swagger 没找到 Authorize 按钮" | 改为用 Bash curl 直接测试 API |
| "你一定要我来操作吗，自己操作不行吗" | 之后所有 API 测试由 AI 直接执行，不再让用户手动粘贴命令 |
| "上传后列表没立即显示，要刷新" | 先改 `:key` 强制重建，后改为 docId 立即返回 |
| "删除文档页面没反应" | 修 204 无 body 时 api.delete 报错 |
| "选择文档后一直提示上传中" | 改异步处理，秒回 201，后台处理 |
