# 测试策略

本文档描述测试架构、分层与覆盖范围、运行方式和编写新测试的规范。

---

## 测试分层

| 层级 | 位置 | 框架 | 数据库 | 覆盖范围 | 运行命令 |
|------|------|------|--------|---------|---------|
| 单元测试 | `src/__tests__/` | Vitest | — | Pipeline 函数、Service 逻辑、工具函数、配置校验 | `npm test` |
| 集成测试 | `test/` | Vitest | `ragtest`（独立库） | API 端点、数据库操作、中间件 | `npm test` |
| E2E 测试 | `test/e2e/` | Playwright | 共享开发库 | 关键用户流程 | `npm run test:e2e` |

> 单元测试和集成测试均由 `npm test` 执行，通过 `vitest.config.ts` 统一配置。

## 单元测试

### 文件清单

```
src/__tests__/
├── config.test.ts              # Zod 配置校验
├── lib/
│   ├── errors.test.ts          # 错误类 + 错误处理器
│   └── vectordb.test.ts        # LanceDB 向量操作 (V13)
├── middleware/
│   └── auth.test.ts            # 认证/授权中间件
├── pipeline/
│   ├── agent.test.ts           # Agent 函数调用
│   ├── embedder.test.ts        # Embedding 调用
│   ├── generator.test.ts       # LLM 生成
│   ├── reranker.test.ts        # 精排逻辑
│   └── splitter.test.ts        # 文本切分
└── services/
    ├── auth.service.test.ts    # 注册/登录
    ├── chat.service.test.ts    # 问答流程编排
    ├── context.service.test.ts # 对话管理
    └── kb.service.test.ts      # KB CRUD
```

### 特点

- 纯逻辑测试，不依赖 PostgreSQL/Redis/Ollama
- 外部依赖通过 mock 隔离
- 跑在 vitest 默认 jsdom 或 node 环境

## 集成测试

### 架构

`test/setup.ts` 负责测试环境初始化：

1. **清理旧表**：DROP 全部 11 张表（级联）
2. **重建表**：执行 `CREATE TABLE IF NOT EXISTS`（跳过 drizzle migrate，直接 SQL）
3. **构建 App**：调用 `buildApp()` 并 `app.ready()`，导出 `getApp()`
4. **清理**：`afterAll` 中关闭 app

### 测试文件

| 文件 | 覆盖内容 |
|------|---------|
| `auth.test.ts` | 注册 → 登录 → JWT 认证 → 角色校验 |
| `kb.test.ts` | 知识库 CRUD：创建/列表/详情/编辑/删除 + 权限 |
| `chat.test.ts` | 问答流程：创建对话 → 提问 → 检索 → 回答 |
| `admin.test.ts` | 管理员：用户管理、KB 管理、审计日志 |
| `feedback.test.ts` | 消息反馈：赞/踩、切换、统计 |
| `apikey.test.ts` | API Key：生成、鉴权、吊销 |
| `v12-api.test.ts` | V12：趋势图表、失败文档、重试、处理进度 (7 测试) |
| `v13-api.test.ts` | V13：AI 引擎配置、模型列表、向量迁移、基准测试 (6 测试) |

### 运行方式

```bash
# 需要 PostgreSQL 和 Redis 在线
npm test

# Watch 模式
npm run test:watch

# 运行单个文件
npx vitest run test/auth.test.ts
```

### 关键约定

- 测试用独立数据库 `ragtest`（见 `vitest.config.ts` 中 `DATABASE_URL`）
- 测试超时 300 秒（Ollama 推理较慢）
- 使用 `app.inject()` 模拟 HTTP 请求，不走网络层
- 测试不依赖 LanceDB（向量操作在集成测试中 mock 或跳过）
- 每个测试文件独立运行，不依赖执行顺序

## E2E 测试

### 覆盖流程

`test/e2e/rag-flow.spec.ts`：

1. 注册新用户
2. 登录
3. 创建知识库
4. 上传文档
5. 提问并验证回答

### 运行方式

```bash
# 需要前后端均在线
npm run test:e2e
```

### 技术栈

- Playwright（Chromium）
- 自动截图保存在 `test/e2e/screenshots/`

## 编写新测试

### 单元测试模板

```typescript
import { describe, it, expect, vi } from "vitest";

describe("splitter", () => {
  it("应正确切分段落", () => {
    const result = splitText("段落1\n\n段落2\n\n段落3", 500, 50);
    expect(result).toHaveLength(3);
    expect(result[0].content).toBe("段落1");
  });
});
```

### 集成测试模板

```typescript
import { describe, it, expect } from "vitest";
import { getApp } from "./setup.js";

describe("新功能模块", () => {
  it("应返回预期结果", async () => {
    const app = getApp();

    const res = await app.inject({
      method: "POST",
      url: "/api/xxx",
      headers: { Authorization: "Bearer <token>" },
      payload: { key: "value" },
    });

    expect(res.statusCode).toBe(200);
    const body = res.json();
    expect(body.success).toBe(true);
  });
});
```

### 获取测试 Token

```typescript
// 先注册后登录获取 token
const registerRes = await app.inject({
  method: "POST",
  url: "/api/auth/register",
  payload: { username: "testuser", password: "testpass123" },
});

const loginRes = await app.inject({
  method: "POST",
  url: "/api/auth/login",
  payload: { username: "testuser", password: "testpass123" },
});
const token = loginRes.json().data.token;
```

### 注意事项

- 测试间通过 `setup.ts` 的 beforeAll 保证数据库干净
- 不要在测试中硬编码用户 ID，通过 API 响应获取
- Ollama 相关的测试可能需要更长的超时时间
- E2E 测试确保前后端均在线后运行
