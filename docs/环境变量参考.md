# 环境变量参考

> 完整的 `.env` 配置字典。部署或开发时参考此文档。

---

## 必需配置（首次部署必须设置）

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `DATABASE_URL` | string | `postgresql://raguser:ragpass@localhost:5432/ragdb` | PostgreSQL 连接串 |
| `REDIS_URL` | string | `redis://localhost:6379` | Redis 连接串 |
| `OLLAMA_URL` | string | `http://localhost:11434` | Ollama 服务地址 |
| `JWT_SECRET` | string | `change-me-to-a-random-secret` | JWT 签名密钥，**生产环境必须修改**为随机字符串 |
| `UPLOAD_DIR` | string | `./data/uploads` | 上传文档的存储目录 |

---

## LLM 配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `LLM_PROVIDER` | `"ollama" \| "deepseek"` | `ollama` | LLM 来源。`ollama`=本地 CPU 推理（慢但免费），`deepseek`=云端 API（秒级，需配置 API 密钥） |
| `LLM_MODEL` | string | `qwen2.5:7b` | Ollama 模型名，仅在 `LLM_PROVIDER=ollama` 时生效 |
| `DEEPSEEK_API_KEY` | string | `(空)` | DeepSeek API 密钥，仅在 `LLM_PROVIDER=deepseek` 时必需 |
| `DEEPSEEK_MODEL` | string | `deepseek-chat` | DeepSeek 模型名，仅在 `LLM_PROVIDER=deepseek` 时生效 |

**切换示例：**

```bash
# 本地模式（默认）
LLM_PROVIDER=ollama
LLM_MODEL=qwen2.5:7b

# 云端模式（需先获取 DeepSeek API Key）
LLM_PROVIDER=deepseek
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
DEEPSEEK_MODEL=deepseek-chat
```

## Embedding 配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `EMBEDDING_MODEL` | string | `bge-m3` | Ollama Embedding 模型名 |
| `EMBEDDING_DIM` | integer | `1024` | Embedding 向量维度。**必须与模型实际输出匹配**。换模型时需同步修改 |

---

## 检索参数

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `RETRIEVAL_TOP_K` | integer | `25` | 向量/BM25 各检索的前 K 个候选数 |
| `RRF_K` | integer | `60` | RRF 融合算法的 k 参数。控制排名对最终分数的影响程度 |
| `FINAL_TOP_K` | integer | `10` | 最终送入 LLM 上下文的最大片段数 |
| `RERANKER_TOP_K` | integer | `25` | Reranker 候选池大小，仅在 `RERANKER_ENABLED=true` 时生效 |
| `RERANKER_ENABLED` | boolean | `false` | 是否启用精排。开启后对 `RERANKER_TOP_K` 个候选逐条 LLM 评分，能提高准确性但显著降低速度 |
| `SIMILARITY_THRESHOLD` | number (0-1) | `0.5` | 向量搜索结果的最低相似度阈值，低于此值的片段会被过滤 |

---

## 文档处理参数

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `CHUNK_SIZE` | integer | `500` | 文本切分的块大小（字符数） |
| `CHUNK_OVERLAP` | integer | `50` | 相邻块之间的重叠字符数。防止关键信息被切断 |
| `MAX_FILE_SIZE_MB` | integer | `50` | 上传文件大小上限（MB） |
| `INCREMENTAL_INDEX` | boolean | `false` | 是否启用增量索引。开启后重新上传文件只处理内容变化的部分 |

---

## JWT 认证

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `JWT_ACCESS_EXPIRY` | duration | `15m` | Access token 有效期（支持 `s`/`m`/`h`/`d`） |
| `JWT_REFRESH_EXPIRY` | duration | `7d` | Refresh token 有效期 |

---

## Agent

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `AGENT_ENABLED` | boolean | `false` | 是否启用 Agent 能力（意图检测 + 时间查询 + 计算器）。开启后 LLM 回答前先检查是否可被工具调用替代 |

---

## 配置优先级

1. 系统环境变量（最高优先级）
2. `.env` 文件（项目根目录）
3. 代码默认值（最低优先级）

> 环境变量已存在时不会被 `.env` 文件覆盖。`.env` 文件不可提交到 Git（已在 `.gitignore` 中排除）。
