## 1. 这个产品是干什么的

简单说：**你把团队文档传上去，然后像聊天一样问问题，系统自动从文档里找答案。**

和 ChatGPT 的区别：

| | ChatGPT | 本产品 |
|------|---------|--------|
| 数据来源 | 互联网（不知道你公司的内部规定） | 你上传的文档（只回答文档里的内容） |
| 会不会编造 | 可能会瞎编 | 找不到就说"未找到"，不瞎编 |
| 答案追溯 | 不知道答案哪来的 | 每条回答附引用来源，可以点开看原文 |

典型使用场景：
- "公司的年假政策是什么？" → 自动从《员工手册》里找到对应条款
- "OpenLayers 怎么加载 GeoJSON？" → 从技术文档里找到示例代码
- "项目 A 的架构是怎么设计的？" → 从设计文档里检索相关内容

## 2. 基础概念

| 术语 | 大白话解释 |
|------|-----------|
| 知识库 | 一个"文件夹"，放一组相关文档。比如"技术文档库"放技术资料，"人事制度库"放公司规章 |
| 文档 | 你上传的 PDF/Word/Markdown/TXT 文件 |
| 检索 | 系统在你上传的文档中搜索相关内容的过矨 |
| 来源 | 答案引用了哪篇文档的哪个段落。点开可以看原文验证 |
| 对话 | 一次问答会话。追问时系统知道你之前在聊什么 |
| Embedding | 把文字转成数学向量，用来做语义搜索。你不用懂，系统自动处理 |

## 3. 注册与登录

1. 打开浏览器访问系统地址（例如 `http://localhost:5173` 或部署后的域名）
2. 首次使用点"注册"，输入用户名和密码
3. 注册成功后自动登录，进入知识库列表页

> 如果系统管理员给了你账号，直接登录即可。
> 系统首次启动时会自动创建一个管理员账号：`admin` / `admin123`。如果你是第一个使用者，用这个登录后可以新建知识库。

---

## 4. 创建你的第一个知识库

1. 登录后进入首页（知识库列表）
2. 点击右上角 **"+ 新建知识库"** 按钮
3. 输入名称（比如"技术文档库"），描述可选
4. 点击"创建"
5. 页面自动跳转到该知识库

一个知识库就像一个分类文件夹，建议按主题创建：
- "公司制度" → 员工手册、考勤规定、报销流程
- "技术文档" → 架构设计、API 文档、故障报告
- "产品资料" → PRD、用户手册、竞品分析

---

## 5. 上传文档

1. 进入知识库，点击顶部 **"文档管理"** 标签
2. 点击 **"+ 上传文档"** 按钮
3. 选择要上传的文件（支持 PDF、Word、Markdown、TXT、Excel、PPT）
4. 上传后文档先显示"处理中"
5. 等待 1-2 分钟（大文件更长），状态变为"就绪"即可使用

> **注意**：
> - 文件名建议用中文或英文描述清楚，它会出现在答案的来源引用里
> - 文档中的表格、图片可能无法完美解析，纯文本内容最佳
> - 一次可以上传多个文件，处理是并行的

## 6. 提问与对话

### 基础提问

1. 进入知识库，默认在 **"问答"** 标签
2. 在下方的输入框输入问题，按回车或用鼠标点"发送"
3. 等待片刻，系统返回答案

**好问题的特征**：
- 明确具体："年假有多少天？" 比 "休假" 好
- 有上下文："2025 年版本的考勤规定是什么？" 比 "考勤" 好

### 引用来源

每条回答下面会显示 **"引用来源 (N)"**，点击展开可以看到：
- 来自哪篇文档
- 原文片段
- 匹配相似度

这让你可以快速验证答案是否准确——如果原文确实那么写的，答案可信；如果发现文不对题，说明检索不够精准，换个问法试试。

### 答案置信度（V12 新增）

每条 AI 回答的左上角会显示置信度标签：

- **高置信度（绿色）**：最佳匹配来源相似度 >= 70%，答案可信度高
- **中置信度（黄色）**：相似度 40%~70%，建议点开来源确认
- **低置信度（灰色）**：相似度 < 40%，答案可能不够准确，建议换个问法或补充文档

### 引用编号（V10 新增）

AI 回答中会出现可点击的 `[1]` `[2]` 编号，对应具体的引用来源：

- **点击编号**：消息下方展开该来源的完整原文片段
- **再次点击**：收起原文
- 这个功能让你不用翻到底部就能快速查看某一条证据

### 多轮对话

你可以像和人聊天一样追问：

```
你: "年假有多少天？"
AI: "工作满一年享有5天年假。（来源：考勤制度.md）"

你: "那婚假呢？"
AI: "员工结婚享有3天婚假。（来源：考勤制度.md）"
```

系统会自动理解"那婚假呢"指的是"公司的婚假政策"，不需要你每次都说完整句子。

### 没找到答案怎么办

如果系统返回 **"未找到相关内容"**，试试：

1. **换个关键词**："年假" 改成 "带薪休假"
2. **换个问法**："年假怎么算" 改成 "休假政策"
3. **确认文档已上传**：你想问的内容在已上传的文档里吗？
4. 联系管理员：可能文档还没处理完，或者知识库内容需要补充

## 7. 查看对话历史

1. 在问答页面的**左侧边栏**可以看到"对话历史"
2. 点击任意一条历史对话，可以在右侧加载当时的完整问答案
3. 点击"+ 新对话"开始新的会话
4. 每条对话旁边的 ⬇ 按钮可以导出为 Markdown 文件
5. 不想要的对话点进去可以删除

### 搜索对话（V10 新增）

对话历史列表顶部有搜索框，输入关键词即可实时过滤对话标题，方便在大量对话中快速找到目标。

搜索时列表不会自动刷新（暂停轮询），清空搜索框后恢复。

## 8. 管理文档

1. 进入知识库，点击 **"文档管理"** 标签
2. 可以看到所有已上传的文档列表
3. 每行显示：文件名、类型、处理状态、文本块数
4. 就绪的文档会出现 **"预览"** 按钮（V10），点击弹窗展示完整文档内容
5. 点击"删除"可以移除不再需要的文档

**状态说明**：

| 状态 | 含义 |
|------|------|
| 排队中 | 文档已上传，等待处理 |
| 处理中 | 正在解析和索引，请稍候 |
| 就绪 | 处理完成，可以提问了 |
| 失败 | 处理出错（可能是文件损坏或格式不支持），删除重试 |

---

## 9. 答案反馈（V11 新增）

每条 AI 回答的右下角有 👍 和 👎 按钮：

- **点赞（👍）**：表示这个回答有用、准确
- **点踩（👎）**：表示这个回答有问题
- **再次点击**：取消之前的评价
- 反馈数据会被管理员在管理控制台的"反馈概览"中查看，帮助优化知识库质量

## 10. API Key 与设置（V11 新增）

### 什么是 API Key

API Key 是一个密钥，让你可以在不登录网页的情况下通过程序调用问答接口。适合自动化脚本、CI/CD 集成等场景。

### 如何管理

1. 点击顶栏的 **"设置"** 链接
2. 输入一个名称（方便区分用途），点击"生成 API Key"
3. **立即复制密钥**——它只显示一次，关闭页面后无法找回
4. 不再使用的 Key 可以点击"吊销"

### 使用方式

```bash
curl -X POST http://localhost:3000/api/kb/{知识库ID}/chat \
  -H "Authorization: Bearer ak_xxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{"question": "你的问题"}'
```

> API Key 和网页登录互不影响，吊销 Key 不会影响你的登录状态。

## 11. 使用技巧

### 让回答更准确

1. **文档标题要有意义**：`考勤管理制度.md` 比 `文档1.docx` 好，因为标题会出现在引用来源里
2. **结构化写作**：用 Markdown 的标题 `#` `##` 分隔章节，系统切分时会保留段落结构
3. **避免超大文件**：单文件超过 1MB 的建议拆分成多个小文件，处理和检索都更快
4. **关键信息不要只放在表格里**：当前版本表格解析不完美，把关键内容也写在正文里
5. **一个知识库一个主题**：不要把人事制度和产品文档混在一个知识库里，会影响检索精度

### 提高效率

1. 先上传少量文档试用，确认效果后再批量导入
2. 每次上传后等到状态变"就绪"再提问
3. 如果某类问题反复回答不好，考虑补充相关文档

## 12. 常见问题

### Q: 上传的文档多久能处理完？

**A**：小文件（几十 KB）1-2 分钟，大文件可能 5-10 分钟。取决于服务器性能。处理是异步的，上传后可以去做别的。

### Q: 能上传多大的文件？

**A**：单个文件不超过 50MB。超过的话拆分成多个小文件。

### Q: 支持哪些文件格式？

**A**：PDF、Word (.docx)、Excel (.xlsx)、PPT (.pptx)、Markdown (.md)、纯文本 (.txt) 共 6 种格式。不支持图片（图片里的文字无法识别）。

### Q: 为什么有些问题回答得很准确，有些完全不对？

**A**：检索质量取决于三个因素：
1. 文档里是否真的有答案（没有的话系统没法凭空回答）
2. 你的问题是否和文档里的表述接近（太口语化可能搜不到）
3. 文档是否已经处理完（状态为"就绪"）

### Q: 我能看到别人上传的文档吗？

**A**：当前版本所有登录用户共享知识库里的文档。如果不想让别人看到某些文档，建议单独建一个知识库，由管理员控制访问。严格的文档级权限会在后续版本加入。

### Q: 回答里面的信息能保证 100% 准确吗？

**A**：不能。系统**尽量不编造**，但 LLM 偶尔会错误理解文档内容。建议：如果答案涉及重要决策，点开源来源的原文片段做确认。

### Q: 能问一些和文档无关的问题吗？

**A**：如果管理员开启了 Agent 功能，系统可以回答部分通用问题。例如：
- "现在几点？" → 返回当前时间
- "100 乘以 50 等于多少？" → 返回计算结果

这类问题不走文档检索，直接由 Agent 处理并返回结果。
