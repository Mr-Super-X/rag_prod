# 数据库设计文档

> 面向开发者、运维人员。完整的表结构、字段说明、索引策略和外键关系。

---

## 概述

- **数据库**：PostgreSQL 15
- **ORM**：Drizzle ORM（`src/db/schema.ts`）
- **表数量**：11 张
- **主键策略**：全部使用 `UUID DEFAULT gen_random_uuid()`
- **时间戳**：全部使用 `TIMESTAMPTZ DEFAULT now()`

---

## ER 关系图

```
users (1) ────────< knowledge_bases (N)     createdBy → users.id
users (1) ────────< kb_members (N)          userId → users.id
users (1) ────────< documents (N)           uploadedBy → users.id
users (1) ────────< conversations (N)       userId → users.id
users (1) ────────< refresh_tokens (N)      userId → users.id → CASCADE
users (1) ────────< audit_logs (N)          userId → users.id → CASCADE
users (1) ────────< api_keys (N)            userId → users.id → CASCADE
users (1) ────────< message_feedback (N)    userId → users.id → CASCADE

knowledge_bases (1) ═< kb_members (N)       kbId → kb.id → CASCADE
knowledge_bases (1) ═< documents (N)        kbId → kb.id → CASCADE
knowledge_bases (1) ═< chunks (N)           kbId → kb.id → CASCADE
knowledge_bases (1) ═< conversations (N)    kbId → kb.id → CASCADE

documents (1) ═< chunks (N)                 docId → docs.id → CASCADE

conversations (1) ═< messages (N)           conversationId → conv.id → CASCADE

messages (1) ═< message_feedback (N)        messageId → msg.id → CASCADE
```

> ═ 表示 CASCADE 删除（删除父记录时自动删除子记录）\
> ── 表示 RESTRICT（子记录存在时不可删除父记录，先删子表）

## 表详细说明

### 1. users — 用户

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK, DEFAULT gen_random_uuid() | 用户唯一标识 |
| `username` | VARCHAR(50) | NOT NULL, UNIQUE | 用户名 |
| `password_hash` | VARCHAR(255) | NOT NULL | bcrypt 哈希后的密码 |
| `role` | VARCHAR(20) | NOT NULL, DEFAULT "user" | 角色：`admin` 或 `user` |
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 创建时间 |

---

### 2. knowledge_bases — 知识库

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | 知识库唯一标识 |
| `name` | VARCHAR(200) | NOT NULL | 名称 |
| `description` | TEXT | — | 描述 |
| `created_by` | UUID | NOT NULL, FK → users.id | 创建者 |
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 创建时间 |
| `updated_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 更新时间 |
| `migration_status` | VARCHAR(20) | NOT NULL, DEFAULT 'ready' | V13: 向量迁移状态 |
| `vector_version` | VARCHAR(20) | NOT NULL, DEFAULT '' | V13: 活跃向量表版本后缀 |

---

### 3. kb_members — 知识库成员

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | — |
| `kb_id` | UUID | NOT NULL, FK → knowledge_bases.id CASCADE | 所属知识库 |
| `user_id` | UUID | NOT NULL, FK → users.id CASCADE | 成员用户 |
| `role` | VARCHAR(20) | NOT NULL, DEFAULT "member" | 角色：`owner` 或 `member` |
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 加入时间 |

**索引**：`idx_kb_members_kb_id`(kb_id), `idx_kb_members_user_id`(user_id)

---

### 4. documents — 文档

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | 文档唯一标识 |
| `kb_id` | UUID | NOT NULL, FK → knowledge_bases.id CASCADE | 所属知识库 |
| `filename` | VARCHAR(500) | NOT NULL | 原始文件名 |
| `file_type` | VARCHAR(20) | NOT NULL | 扩展名(pdf/docx/md/txt/xlsx/pptx) |
| `file_size` | BIGINT | — | 文件大小（字节） |
| `storage_path` | VARCHAR(500) | NOT NULL | 服务端存储路径 |
| `status` | VARCHAR(20) | NOT NULL, DEFAULT "processing" | 处理状态 |
| `chunk_count` | INT | DEFAULT 0 | 切分后的 chunk 数量 |
| `uploaded_by` | UUID | NOT NULL, FK → users.id | 上传者 |
| `error_message` | TEXT | — | 处理失败时的错误信息 |
| `progress_step` | VARCHAR(20) | V12: parsing/splitting/embedding/indexing/ready | 处理分步进度 | 
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 创建时间 |
| `updated_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 更新时间 |

**status 状态机**：`processing` → `ready` / `error`

**索引**：`idx_documents_kb_id`(kb_id)

---

### 5. chunks — 文本片段

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | — |
| `doc_id` | UUID | NOT NULL, FK → documents.id CASCADE | 所属文档 |
| `kb_id` | UUID | NOT NULL, FK → knowledge_bases.id CASCADE | 所属知识库（冗余，加速查询） |
| `qdrant_point_id` | VARCHAR(100) | NOT NULL | 对应 LanceDB 中的向量点 ID |
| `content` | TEXT | NOT NULL | 文本内容 |
| `chunk_index` | INT | NOT NULL | 在文档中的顺序号 |
| `metadata` | JSONB | DEFAULT '{}' | 扩展元数据 |
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 创建时间 |

**索引**：`idx_chunks_doc_id`(doc_id), `idx_chunks_kb_id`(kb_id), `idx_chunks_qdrant_point_id`(qdrant_point_id)

---

### 6. conversations — 对话

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | 对话唯一标识 |
| `kb_id` | UUID | NOT NULL, FK → knowledge_bases.id CASCADE | 所属知识库 |
| `user_id` | UUID | NOT NULL, FK → users.id | 对话发起者 |
| `title` | VARCHAR(300) | — | 对话标题（首问截取） |
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 创建时间 |
| `updated_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 最后活动时间 |

**索引**：`idx_conversations_user_id`(user_id), `idx_conversations_kb_id`(kb_id)

---

### 7. messages — 消息

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | — |
| `conversation_id` | UUID | NOT NULL, FK → conversations.id CASCADE | 所属对话 |
| `role` | VARCHAR(20) | NOT NULL | `user` 或 `assistant` |
| `content` | TEXT | NOT NULL | 消息正文 |
| `sources` | JSONB | — | 引用来源数组（仅 assistant） |
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 创建时间 |
| `updated_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 更新时间 |

**索引**：`idx_messages_conversation_id`(conversation_id)

**sources JSONB 结构**：`[{ chunkId, content, score, docFilename }]`

---

### 8. refresh_tokens — 刷新令牌

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | — |
| `user_id` | UUID | NOT NULL, FK → users.id CASCADE | 所属用户 |
| `token` | VARCHAR(500) | NOT NULL, UNIQUE | 刷新令牌（随机 96 位 hex） |
| `expires_at` | TIMESTAMPTZ | NOT NULL | 过期时间（创建时 +7 天） |
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 创建时间 |

**索引**：`idx_refresh_tokens_token`(token), `idx_refresh_tokens_user_id`(user_id)

---

### 9. message_feedback — 消息反馈

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | — |
| `message_id` | UUID | NOT NULL, FK → messages.id CASCADE | 被评价的消息 |
| `user_id` | UUID | NOT NULL, FK → users.id CASCADE | 评价者 |
| `rating` | INT | NOT NULL | 1=赞, -1=踩 |
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 评价时间 |
| `updated_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 更新时间 |

**索引**：`idx_feedback_message_id`(message_id), `idx_feedback_user_id`(user_id)

**反馈逻辑**：同一用户对同一消息只能有一个评价。再次点击同一按钮取消，切换按钮更新记录。

---

### 10. audit_logs — 审计日志

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | — |
| `user_id` | UUID | NOT NULL, FK → users.id CASCADE | 操作者 |
| `action` | VARCHAR(50) | NOT NULL | 操作类型：login/create_kb/delete_kb/upload_doc/delete_doc |
| `resource` | VARCHAR(50) | NOT NULL | 资源类型：kb/document/user |
| `resource_id` | VARCHAR(100) | — | 资源 ID |
| `ip` | VARCHAR(45) | — | 操作者 IP |
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 操作时间 |

**索引**：`idx_audit_user_id`(user_id), `idx_audit_created_at`(created_at)

---

### 11. api_keys — API 密钥

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID | PK | — |
| `user_id` | UUID | NOT NULL, FK → users.id CASCADE | 所属用户 |
| `key_hash` | VARCHAR(255) | NOT NULL, UNIQUE | HMAC-SHA256(JWT_SECRET, plaintext_key) |
| `key_prefix` | VARCHAR(8) | NOT NULL | 明文密钥前 8 位（用于 UI 展示：`ak_` 前缀 + 掩码） |
| `name` | VARCHAR(100) | NOT NULL | 用户给 Key 起的名称 |
| `last_used_at` | TIMESTAMPTZ | — | 最后使用时间 |
| `created_at` | TIMESTAMPTZ | NOT NULL, DEFAULT now() | 创建时间 |
| `revoked_at` | TIMESTAMPTZ | — | 吊销时间（NULL = 有效） |

**索引**：`idx_api_keys_hash`(key_hash)

**安全设计**：仅存储哈希，创建时一次性返回明文密钥（`ak_` 前缀 + 随机字符串）。吊销操作将 `revoked_at` 设为当前时间，不物理删除。

## 索引策略总结

| 表 | 索引列 | 用途 |
|----|--------|------|
| documents | kb_id | 按知识库查文档列表 |
| chunks | doc_id | 按文档查 chunks（预览/重建） |
| chunks | kb_id | 按 KB 查 chunks |
| chunks | qdrant_point_id | 关联 LanceDB 向量点 |
| conversations | user_id | 按用户查对话列表 |
| conversations | kb_id | 按 KB 查对话列表 |
| messages | conversation_id | 按对话查消息列表 |
| refresh_tokens | token | Token 查找（登录、refresh） |
| refresh_tokens | user_id | 按用户查 tokens |
| kb_members | kb_id | 按 KB 查成员 |
| kb_members | user_id | 按用户查所属 KB |
| audit_logs | user_id | 按用户查日志 |
| audit_logs | created_at | 按时间倒序展示 |
| api_keys | key_hash | API Key 鉴权查找 |
| message_feedback | message_id | 按消息查反馈 |
| message_feedback | user_id | 按用户查反馈 |

## 迁移流程

```bash
# 修改 src/db/schema.ts → 生成 SQL（由 Drizzle Kit 比较 schema 差异自动生成）
npx drizzle-kit generate

# 执行迁移
npx drizzle-kit migrate

# 可视化管理
npx drizzle-kit studio
```

> 迁移文件位于 `src/db/migrations/`，由 Drizzle Kit 自动管理。**不要手动编辑**。
