# 权限模型说明

> 面向开发者、运维人员和安全审计。完整描述平台的用户角色、知识库访问权限、中间件机制和鉴权流程。

---

## 角色体系

### 全局角色（用户级别）

| 角色 | 标识 | 存储 | 说明 |
|------|------|------|------|
| admin | `users.role = "admin"` | PostgreSQL | 系统管理员。可管理所有用户和知识库 |
| user | `users.role = "user"` | PostgreSQL | 普通用户。可创建和管理自己的知识库 |

### 知识库角色（KB 级别）

| 角色 | 标识 | 存储 | 说明 |
|------|------|------|------|
| owner | `kb_members.role = "owner"` | PostgreSQL | KB 创建者。可管理 KB 设置、成员、文档。不能被移除 |
| member | `kb_members.role = "member"` | PostgreSQL | 普通成员。可访问 KB、上传文档、提问 |

## 鉴权中间件

### 1. authenticate

验证请求中的 JWT access token。

```
HTTP 请求 → Authorization: Bearer <access_token>
           → jwtVerify() → 失败返回 401
           → 成功：注入 request.user{ id, username, role }
```

### 2. requireAdmin

检查当前用户是否为 admin。

```
authenticate 后 → request.user.role === "admin"
                → 否 → 返回 403
```

### 3. requireKBAccess

检查当前用户是否有访问目标知识库的权限。

```
authenticate 后 → 查 knowledge_bases.createdBy === userId → 是 → 放行
                → 查 kb_members[userId, kbId] 存在 → 是 → 放行
                → 否 → 返回 403
```

### 4. authenticateApiKey

验证 API Key（替代 JWT 的鉴权方式）。

```
Authorization: Bearer ak_xxxx
  → 识别前缀 "ak_" → 查 api_keys 表（HMAC-SHA256 哈希比对）
  → 检查 revokedAt 为空
  → 注入 request.user → 交给后续中间件
  → 匹配失败 → 静默放行（不报错，让 authenticate 继续处理）
```

## 路由权限矩阵

| 路由 | 认证 | 权限要求 | 说明 |
|------|------|---------|------|
| `POST /api/auth/register` | 无 | — | 公开注册 |
| `POST /api/auth/login` | 无 | — | 公开登录 |
| `POST /api/auth/refresh` | 无 | — | Token 刷新 |
| `GET /api/kb` | authenticate | — | 列出自己的知识库 |
| `POST /api/kb` | authenticate | — | 创建知识库（自动成为 owner） |
| `GET /api/kb/:id` | authenticate + requireKBAccess | KB owner 或 member | 查看 KB 详情 |
| `PATCH /api/kb/:id` | authenticate | KB owner | 编辑 KB |
| `DELETE /api/kb/:id` | authenticate | KB owner 或 admin | 删除 KB |
| `POST /api/kb/:id/docs` | authenticate + requireKBAccess | KB owner 或 member | 上传文档 |
| `GET /api/kb/:id/docs` | authenticate + requireKBAccess | KB owner 或 member | 文档列表 |
| `DELETE /api/kb/:id/docs/:docId` | authenticate + requireKBAccess | KB owner 或 member | 删除文档 |
| `GET /api/kb/:id/docs/:docId/preview` | authenticate + requireKBAccess | KB owner 或 member | 文档预览 |
| `POST /api/kb/:id/chat` | authenticate + requireKBAccess | KB owner 或 member，也支持 API Key | 问答 |
| `GET /api/kb/:id/conversations` | authenticate + requireKBAccess | KB owner 或 member | 对话列表 |
| `GET /api/kb/:id/members` | authenticate + requireKBAccess | KB owner 或 member | 成员列表 |
| `POST /api/kb/:id/members` | authenticate + requireKBAccess | KB owner 或 member | 添加成员 |
| `DELETE /api/kb/:id/members/:userId` | authenticate + requireKBAccess | KB owner 或 member（不可移除 owner 角色成员） | 移除成员 |
| `GET /api/conversations/:id` | authenticate | 对话所有者 | 对话详情 |
| `DELETE /api/conversations/:id` | authenticate | 对话所有者 | 删除对话 |
| `GET /api/conversations/:id/export` | authenticate | 对话所有者 | 导出对话 |
| `POST /api/messages/:id/feedback` | authenticate | 消息所属对话的所有者 | 赞/踩反馈 |
| `GET /api/admin/users` | authenticate + requireAdmin | admin | 用户列表 |
| `DELETE /api/admin/users/:id` | authenticate + requireAdmin | admin | 删除用户 |
| `GET /api/admin/kbs` | authenticate + requireAdmin | admin | KB 列表（含统计） |
| `DELETE /api/admin/kbs/:id` | authenticate + requireAdmin | admin | 强制删除 KB |
| `GET /api/admin/overview` | authenticate + requireAdmin | admin | 使用概览 |
| `GET /api/admin/audit-logs` | authenticate + requireAdmin | admin | 审计日志 |
| `GET /api/admin/feedback-stats` | authenticate + requireAdmin | admin | 反馈统计 |
| `POST /api/auth/api-keys` | authenticate | — | 生成 API Key |
| `DELETE /api/auth/api-keys/:id` | authenticate | — | 吊销 API Key |
| `GET /api/health` | 无 | — | 健康检查 |

## 鉴权调用链

```
请求进入
  │
  ├─ /api/kb/:id/chat（支持双鉴权）
  │     └─ app.addHook("onRequest", authenticateApiKey) → 尝试 API Key
  │     └─ app.addHook("onRequest", authenticate)       → JWT 校验（API Key 已通过则跳过）
  │     └─ app.addHook("onRequest", requireKBAccess)     → KB 成员检查
  │
  ├─ /api/admin/*（管理员路由）
  │     └─ app.addHook("preHandler", authenticate)       → JWT 校验
  │     └─ app.addHook("preHandler", requireAdmin)       → admin 检查
  │
  └─ /api/kb（普通用户路由）
        └─ app.addHook("onRequest", authenticate)        → JWT 校验
```

## 特殊规则

| 规则 | 说明 |
|------|------|
| KB 创建者始终有权限 | `knowledge_bases.createdBy === userId` 直接放行，不查 `kb_members` 表。兼容 V4 之前的旧数据 |
| 成员不可移除 owner | `DELETE /api/kb/:id/members/:userId` 拒绝移除 `kb_members.role === "owner"` 的成员 |
| admin 可强制删除 KB | `DELETE /api/admin/kbs/:id` 绕过 `requireKBAccess`，直接删库（级联删除所有文档/对话/成员） |
| API Key 哈希存储 | `api_keys.keyHash` = HMAC-SHA256(JWT_SECRET, plaintext_key)，永远不存储明文 |
| Refresh Token rotation | 每次 refresh 用旧 token 换取新 token，旧 token 立即删除。防止长期持有一个 token 的风险 |
