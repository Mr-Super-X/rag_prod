# 技术栈与关键名词速查

> 5 分钟看懂这个项目用了什么技术、每个词是什么意思。不用懂编程也能读。

---

## 一、打个比方

这个项目就像一个**带图书馆的智能客服**：

1. 你把公司的文档（PDF、Word等）上传到图书馆
2. 图书馆管理员（程序）把每本书拆成一页页，贴上标签，按内容分类
3. 你来问问题，管理员去图书馆找出最相关的几页
4. 把这几页内容交给 AI 阅读，AI 根据这些内容回答你
5. 回答里会标出 [1] [2] 编号，点击可以看原文出处

## 二、技术栈一张表

| 技术 | 一句话解释 | 在项目里干什么 |
|------|-----------|--------------|
| TypeScript | 带"类型标签"的 JavaScript，写错了会提前报错 | 前后端全部代码 |
| Node.js | 让 JavaScript 能在服务器上跑 | 后端运行环境 |
| Fastify | Node.js 的 Web 框架，比 Express 快 2-3 倍 | 处理用户请求，返回结果 |
| PostgreSQL | 老牌关系型数据库，像 Excel 表格 | 存储用户、文档、对话等数据 |
| Redis | 内存数据库，读写极快（微秒级） | 文档关键词索引 + 问答缓存 |
| Vue 3 | 前端框架，把页面拆成独立组件 | 所有页面（登录、知识库、管理后台） |
| Pinia | Vue 的状态管理，组件之间共享数据 | 用户登录状态 |
| Vite | 前端开发工具，热更新秒级 | 前端编译打包 |
| Drizzle ORM | 用 TypeScript 代码操作数据库，不用写 SQL | 读写 PostgreSQL |
| Ollama | 一键运行 AI 模型的工具 | 本地运行大语言模型和文档向量化模型 |
| LanceDB | 嵌入式向量数据库，数据存本地文件 | 存储文档的"数字指纹"（向量） |
| Docker Compose | 一键启动多个服务的工具 | 启动 PostgreSQL + Redis + Ollama |
| JWT | 一串加密的"通行证"，证明你是谁 | 用户登录认证 |
| SSE | 服务器主动推送数据，像直播弹幕 | AI 回答逐字流式输出 |
| ECharts | 百度开源图表库 | Admin 趋势图折线 |

## 三、AI 相关名词

| 名词 | 一句话 | 项目里用的哪个 |
|------|--------|--------------|
| **LLM（大语言模型）** | 会聊天的 AI，能读文字回答问题 | Qwen2.5-7B（阿里开源，本地运行）/ DeepSeek（云端） |
| **Embedding（向量化）** | 把一段话变成一串数字（向量），意思相近的话数字也相近 | BGE-M3（BAAI 开源） |
| **RAG（检索增强生成）** | AI 回答问题前先去知识库查资料，用资料回答而不是瞎编 | 这个项目的核心 |
| **Reranker（精排）** | 对 AI 查到的资料做二次排序，把最相关的排最前面 | LLM 列表式精排 |
| **Agent（智能代理）** | AI 不仅能聊天，还能调工具（查时间、算数学） | 时间查询 + 计算器 |

## 四、核心功能名词

| 名词 | 一句话 |
|------|--------|
| **知识库** | 你上传的文档集合，一个知识库 = 一个项目/团队的全部文档 |
| **Chunk** | 文档被切分后的段落（约 500 字一块），是最小的检索单位 |
| **文档摄入** | 上传文档 → 拆分段 → 向量化 → 存入索引的全过程 |
| **向量检索** | 找"意思最接近"的段落（不是找原文一样的词） |
| **BM25 检索** | 传统关键词搜索，找"包含相同词"的段落 |
| **混合检索** | 向量检索 + BM25 同时用，互补短处 |
| **多轮对话** | AI 记住你之前说的话，"那第二个呢"也能懂 |
| **流式输出** | AI 回答一个字一个字蹦出来，不用等全部写完 |
| **引用溯源** | AI 回答里 [1] [2] 编号，点击展开原始文档原文 |
| **审计日志** | 记录了谁在什么时间做了什么操作 |
| **API Key** | 一串密钥（ak_开头），让程序也能调用问答 API |
| **向量迁移** | 换了一个更好的"数字翻译"模型后，把旧翻译重新做一遍 |
| **答案缓存** | 同一个问题问第二遍，秒回（不用再跑全套流程） |

## 五、从用户视角看版本

| 版本 | 加了什么新功能 |
|------|--------------|
| V1 | 上传文档 + 提问 + AI 回答 |
| V2 | AI 回答逐字输出（不用等） + 登录更安全 |
| V3 | 自动备份 + 日志 + 文档更新只处理新增部分 |
| V4 | 限速（防止刷爆） + 知识库只给有权限的人用 |
| V5 | 可选 DeepSeek 云端 AI（秒回但需付费） + 深色模式 |
| V6 | 管理知识库成员 + AI 能查时间算数学 |
| V7 | Excel 和 PPT 也能上传 + 知识库按"我的/共享的"分类 |
| V8 | 管理后台（管理员能看所有用户和知识库） |
| V9 | 对话导出成 Markdown + 知识库用多少一目了然 |
| V10 | AI 回答的引用编号能点开看原文 + 文档内容预览 |
| V11 | 给 AI 回答点赞/踩 + 所有操作有记录 + API 密钥 |
| V12 | Admin 趋势图 + 回答靠谱度标签 + 文档处理进度 + 失败自动重试 |
| V13 | Admin 面板切换 AI 模型 + 换模型后自动重建向量索引 |
| V14 | AI 回答缓存（重复问题秒回） + 精排加速 |

## 六、文件结构速览

```
rag_prod/
├── src/                    ← 后端代码
│   ├── routes/             ← API 端点（处理用户请求）
│   ├── services/           ← 业务逻辑（问答、文档处理）
│   ├── pipeline/           ← AI 流水线（解析、切分、向量化、生成）
│   ├── lib/                ← 工具库（数据库、缓存、向量库）
│   ├── middleware/         ← 中间件（登录检查、权限校验）
│   ├── db/                 ← 数据库表定义和迁移文件
│   └── __tests__/          ← 单元测试
├── client/                 ← 前端代码
│   └── src/
│       ├── pages/          ← 页面（登录、知识库列表、管理后台）
│       ├── components/     ← 组件（聊天面板、文档列表、趋势图）
│       ├── stores/         ← 状态管理（用户登录）
│       └── lib/            ← 工具（API 请求封装）
├── docs/                   ← 文档（你正在看这里）
├── test/                   ← 集成测试
├── data/                   ← 运行时数据（上传的文件、向量库）
├── docker-compose.yml      ← 一键启动基础设施
├── .env                    ← 配置文件（数据库密码、AI 模型名等）
└── CLAUDE.md               ← AI 编程助手的上下文说明
```
